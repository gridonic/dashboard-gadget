Bachelor Thesis im Studiengang Informatik
Startseite

Interaktives Dashboard Gadget

Zusammenfassung

Eine ergänzende Hardware Komponente zu einem Web Dashboard welche einen neuen Ansatz zur Erleichterung der Büro Organisation darstellt.

Schlüsselbegriffe

3D-Druck, AutoCAD, C++, Arduino, JavaScript, Node.js, Socket.io, MongoDB

Zielsetzung

Die Informationen, welchen den Mitarbeitenden via einem web-basierten Dashboard angezeigt werden, lenken diese zu stark ab. Die Mitarbeitenden müssen jeweils in den Browser wechseln um sich spezifische Informationen beschaffen zu können. Der Auftraggeber stellt sich ein Gadget vor, welches verschiedene Informationen abseits des Screens darstellen kann. Ein Schwerpunkt liegt beim Punkt, dass die jeweiligen Mitarbeitenden über ihr eigenes Gadget spezifische Stati wie ihr eigenes Befinden und die aktuelle und oft ändernde Projektzugehörigkeit anzeigen können. Diese Settings sollen vom Gadget aus gesetzt werden können.

Ausgangslage

Das bestehende Dashboard ist web-basiert und als normale Webseite im Internet zu finden. Für gewisse Inhalte vom Dashboard ist das geeignet, der Mitarbeitende kann diese Informationen bei Bedarf abrufen. Andere Inhalte vom Dashboard wie beispielsweise Server-Stati oder die aktuelle Arbeitszeit sollten regelmässig eingesehen werden können. Die Firma hatte daraufhin die Idee, für solche regelmässige Benachrichtungen oder andauernd ändernde Informationen an ein externes Gerät auszulagern, welches für den Mitarbeiter oder die Mitarbeiterin jederzeit sichtbar ist. So kann sichergestellt werden, dass nicht andauernd das web-basierte Dashboard geöffnet ist sondern dass dieses wirklich nur von Zeit zu Zeit abgerufen werden muss. Auf dem externen Gadget geht der Fokus beim normalen Arbeiten nicht verloren und trotzdem hat man jederzeit Einblick in die wichtigsten Informationen.

Ergebnisse

Mittels einer grösseren Umfrage und langen Gesprächen mit dem Auftraggeber konnten wir analysieren, was heutige Büroangestellte für ihre tägliche Arbeit nutzen und wie man sie dabei unterstützen kann. Dabei ist auch unsere eigenen Arbeitserfahrung in diese Analyse miteingeflossen.
Anhand der Anforderungen, welche wir so definierten, haben wir die Hardware des Gadgets gestaltet und die Einzelteile dafür dreidimensional ausgedruckt.
Die finalen Prototypen werden von je einem Arduino MEGA gesteuert und besitzen jeweils ein 240x320px Display um der nutzenden Person die Interaktion zu erleichtern. Über dieses Display wird der nutzenden Person das Menu sowie auch Informationen zur eigenen Arbeitszeit angezeigt. Die Aussenhülle der Gadgets wird mittels LED's in zwei verschiedenen Farben beleuchtet, um die aktuelle Projektzugehörigkeit und den persönlichen Status der jeweiligen Person für alle anderen Personen im Büro sichtbar zu machen.
Um die Geräte möglichst ergonomisch zu gestalten, haben wir ein Scharnier entwickelt, welches dem Nutzer erlaubt, den Winkel vom Display zu ändern. So kann jeder Mitarbeiter bzw. jede Mitarbeiterin den Winkel so einstellen, dass er für seine Sicht auf das Gadget passt.
Mit dem Gadget kann über zwei Buttons, welche wir selber entwickelt haben, interagiert werden. Die Buttons beinhalten jeweils einen Fotodetektor und reagieren auf Veränderung des Umgebungslichts. Deckt man die Buttons also mit einem Finger kurz ab, werden sie betätigt. Eine physische Berührung der Buttons ist so nicht notwendig.
Die Gadgets werden von einem Node.js Server angesteuert, in welchem eine MongoDB Datenbank zur Verwaltung der Daten eingebaut ist. Die Verbindung zwischen Gadget und Server wird via einer Socket Verbindung sichergestellt.
